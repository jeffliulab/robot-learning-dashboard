# 实验报告模板 · Robot Learning Dashboard

本站每一份实验报告都遵循同一套章节骨架。写新报告时照此填内容；子节多的章节（如「实验框架」「实验分析」）再拆二级标题。首篇实证范例见「Cartpole 平衡实验」。

## 页头

- **标题**：实验名（如「Cartpole 平衡实验」）。
- **摘要**：论文式排版——行首粗体「摘要：」标目、占满正文栏宽、两端对齐（dek 加 `abstract` 类）。一段话讲清 *问题 / 方法 / 结果*，只谈本实验，力求简洁。
- **标签行**：内容里只放空锚点 `<div class="meta" data-exp-tags></div>`，标签由 app.js 从 manifest 的实验条目（route/algo/robot/sim 字段）统一注入——**不要在内容里写死标签**。
- **表 1 · 实验概要**：状态、机器人、技术路线、任务环境、**任务配置**、并行环境数、迭代数、训练耗时、观测 / 动作维度、策略网络（含参数量）、最终指标 等一览。机器人一行须写明自由度 / 受控关节总数与主要执行器类型；公开页只显示公开标识，标题、摘要和正文不得暴露内部实验序号或临时 run 命名。
  - **「任务配置」行为必填**，注明任务定义的来源，三选一：*Isaac Lab 官方预设（未修改）* ／ *官方预设 + 命令行覆盖（逐项列出覆盖了什么）* ／ *自定义 cfg 子类（指向 configs/ 下的类）*。目的：预设任务和自定义任务在页头一眼可辨，读者不必翻到「环境与实验设置」节才知道配置动没动过。
- **图 1**：方法总览示意（如智能体–环境交互回路图）。

## 正文章节

1. **背景与问题定义** —— 一句话任务 + **被控对象示意图**（物理系统：自由度、边界、控制输入，形式化记号标注）+ 三个子重点 **机器人 / 控制 / 目标**（按此顺序：系统是什么 → 能操纵什么 → 要达到什么）+ 问题性质；附「问题的形式化定义」表（MDP 要素：环境 / 智能体 / 状态 / 动作 / 奖励 / 回合 / 目标）。
   - **「机器人」子重点必写关节 / 自由度构成**：低维系统列出每个关节及主动 / 被动关系；高维机器人按部位列数量与命名模式（如双腿 12、躯干 1、双臂 10、手指 14），并说明哪些关节受策略动作控制。不能只写“共 N 个受控关节”而不展开。
   - **「控制」子重点必写**，且须给出**因果直觉链**：控制量如何变成任务行为（如“力矩→腿摆动→步态→前进”或“关节位置目标→执行器力矩→接触节律→速度跟踪”），点明中间没有手写控制器、全部由策略学出；高维动作须说明动作维度如何对应到上述关节。
   - **控制量必须画出来**：在图中给控制量一个明确落点（力画直箭头、力矩画绕关节的弧形箭头 ⟳，标注形式化记号如 F、τ）。控制量低维时可直接画进被控对象示意图（如 Cartpole 的水平力 F）；高维时**另配一张控制示意图**（如 Ant 侧视放大 1 条腿标两处 τ），与讲任务的示意图分开——**任务图（要去哪）与控制图（能操纵什么）不得混为一张图里最抢眼的同一根箭头**。范例见 Ant 实验图 3。
2. **理论基础** —— 三步走：①把第 1 节的问题接入 RL 框架；②方法谱系（该问题为什么落到这一族方法）；③本实验所用算法的机制 + **明确的选型依据段**（必须回答"为什么本实验选它"）。全文用到的理论概念（优势 / TD / GAE / 熵正则 / 损失项名称）在此**一次性定义**，后文只引用不复述。
3. **实验框架**（子节多，拆二级）
   - 3.1 网络结构（Actor-Critic，配架构图 + **实测参数量**）
   - 3.2 观测与动作（观测表 + 动作参数逐项）
   - 3.3 奖励函数（**开头先交代累加规则**——如 Isaac Lab 的 权重 × 控制步长 逐步累加，权重是"每秒得分速率"不是单步得分；再点出奖励构成模式"主目标 + 失败重罚 + 小权重塑形"；再权重图 + 逐项：定义 / 动机 / 若移除）
   - 3.4 回合的终止与复位
   - 3.5 训练流程（一次迭代做什么；理论概念引用第 2 节，不重复解释）
   - 3.6 环境与实验设置 —— 任务环境的组织方式（如 Manager-based workflow 在此交代，不要在第 1 节突兀出现）；硬件 / 软件环境**只列版本、不逐条解释**（用名值行 `kv()`）；Rollout 配置与算法超参**逐项附取值依据**（`pr()`）；评价指标；复现命令（并说明命令与注册配置的关系）。特殊环境项（自制硬件等）才正文点一句 + 附录补充。
4. **实验结果** —— 总体数字（含**真实训练耗时**，注明吞吐口径）+ 曲线总图（编号为"图 N"，figcaption 注明数据来源与未修饰）。
5. **实验分析**（子节多，拆二级）
   - 5.1 学习过程分阶段
   - 5.2 辅助曲线互证
   - 5.3 定量对照——按任务性质选：有精确理论上限的任务（如 Cartpole）与上限对照；无上限的任务（如行走的前进奖励）用**多种子稳定性**（≥3 种子的末段 mean±std 与曲线对比）等替代
   - 5.4 策略行为的定性分析（**真实回放**：`<video>` 短回放 + 抽帧序列图 + 只写画面可见行为的描述；没录回放就不写此节，并在局限性中说明。可选加**耐力测试**：回放时覆盖回合时长——如 `env.episode_length_s=60`，仅评估用——检验长时保持能力，并解释训练回合为何设得短）
6. **局限性** —— 如实列（单种子、无独立评估、未测鲁棒性 等）。
7. **结论与未来工作** —— 收敛判定 + 下一步。
- **附录 · 关于本报告**（标题加 `appendix` 类）—— 理论深度说明、数据来源、**参考文献列表**。文献只列于附录，正文不用 [n] 编号引用。

> 消融与对比实验：**做了才设节**（插在实验分析之后，编号顺延）；没做就整节不出现，只在局限性里提"正式对比需 ≥3 随机种子"。写法（范例见 Ant 实验第 6 节）：设计表（条件 × 覆盖参数 × 种子 × 检验目标）→ 对比曲线图（`[data-rl-compare]`，`scripts/export_compare.py` 导出，同条件同色、种子用线型区分）→ 末段 mean±std 结果表（种子间统计，ddof=1）→ 逐条件分析，结论只从实测统计得出。

## 排版机制

- 一级章节用 `h2.section-title`，编号无前导零（1..N）；二级用 `h3.section-sub`，编号如 3.1；附录标题不带编号、加 `.appendix` 类（小一号、灰一档、带上分隔线）。
- 右侧「本页导航」由 `app.js` 的 `buildToc` 自动收 `h2.section-title` + `h3.section-sub` 按文档序渲染，并显示正文中的章节编号；二级项自动缩进。图表分组标题用 `.chart-group` 类（**不得复用 section-sub**，否则泄入 TOC）。
- **表**：表题用语义化 `<table><caption>表 N · …</caption>`（CSS 置顶显示）；名值型表首列加 `class="k"`，不写内联宽度。
- **图**：一切图（含奖励权重条形图、曲线总图、回放帧序列）都包 `<figure>` + `figcaption>图 N · …`；**图表编号连续，且除页头的图 1 / 表 1 外，每个编号必须在正文中被"如图 N / 见表 N"引用至少一次**。
- **图可点击放大**：正文 `<figure>` 直接子级的示意图 SVG 站内一律支持点击弹出放大浮窗（app.js lightbox，Esc/点击关闭；plotly 曲线图不受影响）——复杂图可放心画细，但缩略状态下主信息仍须可辨。
- **媒体必须预留尺寸**：每个 `<img>` / `<video>` 都要写**实测的** `width`/`height` 属性（图用 `file`/`identify` 读，视频读 `videoWidth/Height`；同一实验的视频尺寸未必一致，逐个量、不许照抄）。CSS 已配 `height:auto` 按纵横比预留占位——不写尺寸，媒体加载完成会把下方内容顶移，TOC 长距离跳转会落空、页面加载也会跳版。页面内容本身是一次性全量渲染（无懒加载），plotly 图格子为固定高度，占位稳定性只欠媒体这一环。
- **网络架构图**：一律用 `assets/net-diagram.js` 的 `RLNet.svg(cfg)` 声明式生成（输入分组括号 → 层序列 → 输出头 μ·logσ·N(μ,σ²) 或 V(s)），不手搓 SVG、不手写坐标；层块内只写该层自身信息（神经元数 / 类型+激活 / 参数量），**出入维度由箭头上的 ℝⁿ 维度流携带，块内不得重复**；**各层与合计参数量由维度实时计算，合计须与正文"实测自检查点"的数字吻合**——对不上就是配置或公式错，禁止改文案凑数。Actor/Critic 末层输出维不同（动作维 vs 1），层序列要分开配。新层类型（Conv/GRU…）往文件内 LAYER_TYPES 注册表加条目即可，排版引擎不用动。
- **奖励权重条形图**：中点 0、右绿左红；权重常跨多个数量级，条长按 **log10|w| 在本图 [min,max] → [10%,100%] 线性映射**，figcaption 必须注明"对数刻度、数值以右列为准"（等权重项条长必须相等）。
- 参数逐项用 `pr(名, 值, 理由)`；标准化名值信息用 `kv(名, 值)`。
- 每轮改动后运行 `node scripts/check_report.js`（编号连续性 / 交叉引用存在性 / 图表引用覆盖），并**人工核对**它打印的引用清单——重排后旧号可能仍是合法号，语义指错只有人能看出来。

## 语言与配图规范

- **语域=教科书**（如 Sutton &amp; Barto 中译本）：正式、准确、通顺，术语先解释再用；**禁止**口语化词汇与对读者的闲聊式旁白（如"想改哪一项就改哪一块""大局已定""广撒网试错""砸到谷底""摸到门路"之类）。用"下降/收敛/达到/表明"等书面动词，不用"跌到/攒/切/过几遍/甩回来"。
- **术语定名表**（首次定义后全文固定，不得换叫法）：策略损失（Loss/surrogate）、价值损失（Loss/value）、熵正则项、优势（advantage，定义后用中文）、控制步（不用"策略步"）、σ（动作标准差，分布记号写 N(均值, σ²)）。
- **配图=学术论文式**：框图用中性配色 + 细边框，不用装饰性色块；标注用形式化记号（动作 a、状态 s / 下一状态 s′、奖励 r、维度 ℝⁿ），不写"（会学的）""（给真相）"这类口语标签；图注是完整的一句书面说明。
- **手绘 SVG 排版纪律**（2026-07-10 立，防文字压住图形；机器检查见 `check_report.js` 第 [8] 项；画**机器人骨架/关节图**的专项细则另见仓根《机器人关节图制作指南.md》）：
  - **先算字宽再定坐标**：一行文字的实际宽度 ≈ 逐字累加——1 个汉字/全角标点 ≈ 1 个字号宽（`.sb`=12px、`.lb`=13px），拉丁字母/数字 ≈ 0.58 字号，上下标（ᵢ、³⁷、′）≈ 0.45，空格 ≈ 0.32；再按 `text-anchor`（start 向右展开 / middle 两边各一半 / end 向左展开）从锚点摊开，得到这行字真正占的横向区间。**凭感觉放长中文标注必翻车**（15 个汉字就是 180px）。
  - **留白纪律**：文字外沿离任何笔画/形状 ≥ 20px；整行连同两端必须落在 viewBox 内、四边留 ≥ 30px 边距。
  - **放不下就加画布**：文字列和图形挤在一起时，第一选择是加宽 viewBox、把文字列整体外移——不许把字硬塞进图形旁边的缝里。
  - **注记分区**：视角/规格类注记放画布顶部或底部的专用留白带；成列的说明文字放左右两侧空栏（参照 Ant 图 3 右栏、G1 图 3 右栏）。
  - **刻意画在形状里的文字**（节点框内的"Actor π"、躯干圆里的"躯干"）机器检查会自动放行"完全落在形状内"的情形；启发式判不了的加 `data-in` 属性显式豁免。
  - 每轮改动后跑 `node scripts/check_report.js`——第 [8] 项会按同一套估宽公式抓"文字压图形/越出画布"；机器线是 4px（防估算误差误报），人排版仍按 20px 留。
- **保留辅助结构**：改语域不等于删内容——有用的表格、图、逐项 plist 一律保留。
- 不引用本机源码行号（如 observations.py:212），保留函数名即可。

## 红线

- **不造假、不硬编码**：数字来自真实 cfg / 真实 run（训练耗时读 events、参数量读 checkpoint）；行为定性描述只能来自真实回放；取不到的数写"未记录"。
- **只谈本实验**：正文不出现"本站模板""本站定位""本站后续实验""本系列""上一实验"这类元叙述；不得把本报告的实验动机或结论建立在其他独立报告上。内部实验序号（如 v0.x）、临时目录名、调试 run 名只可作为本地溯源信息保留在私有记录中，公开报告标题、摘要、正文和图表题注一律使用通用、可复现的公开标识。
- **不重复讲通用常识**：跨实验都一样的标准化内容（软硬件环境、通用超参的教科书式解释）不必每篇重讲；把逐条说明的力气留给每个实验都不同的部分（尤其奖励函数）。重点前置——先把核心结论/构成讲明白，再展开细节。
- **理论定义一次**：同一概念（GAE、clip、KL 调度…）只在第 2 节定义，后文引用"见第 2 节"，不复述。
